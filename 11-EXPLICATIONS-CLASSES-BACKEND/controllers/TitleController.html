TitleController.cs - Controller Titres

COUCHE 4 : CONTROLLERS • CRUD titres avec autorisations

Role — Expose les endpoints REST pour gerer les titres avec autorisations par role.

Endpoints et Autorisations

Methode URL Action Autorisation
GET /api/title Liste tous [Authorize] - Tous
GET /api/title/{id} Details [Authorize] - Tous
POST /api/title Creer [Authorize(Roles = "Admin")]
PUT /api/title/{id} Modifier [Authorize(Policy = "ManagerOrAdmin")]
DELETE /api/title/{id} Supprimer [Authorize(Roles = "Admin")]

Attributs d'Autorisation

[Authorize] — Au niveau de la classe : tous les endpoints necessitent authentification. Le client doit envoyer un token JWT valide.
[Authorize(Roles = "Admin")] — Seuls les Admin peuvent acceder. Si Role = "User", retourne 403 Forbidden.
[Authorize(Policy = "ManagerOrAdmin")] — Policy personnalisee. Accepte Manager OU Admin.
