3.2. Workflow pour instructeurs/formateurs

Préparation des cours

Matériel pédagogique
# Cloner le repository complet
git clone https://github.com/inskillflow/ASPNETCORE-WEB-API-XtraWork.git
cd ASPNETCORE-WEB-API-XtraWork

# Créer des dossiers pour chaque étape
mkdir -p cours/etape-1 cours/etape-2 cours/etape-3

# Extraire chaque étape dans son dossier
git archive etape-1-hash | tar -x -C cours/etape-1/
git archive etape-2-hash | tar -x -C cours/etape-2/
git archive etape-3-hash | tar -x -C cours/etape-3/

# Créer des archives ZIP pour distribution
cd cours
zip -r xtrawork-etape-1.zip etape-1/
zip -r xtrawork-etape-2.zip etape-2/
zip -r xtrawork-etape-3.zip etape-3/

Analyse des différences

Points d'enseignement
# Voir les changements entre étapes
git diff --stat etape-1..etape-2
git diff --stat etape-2..etape-3

# Générer un rapport des modifications
git log --oneline etape-1..etape-2 > changements-etape-2.txt
git log --oneline etape-2..etape-3 > changements-etape-3.txt

# Voir les fichiers ajoutés/supprimés
git diff --name-status etape-1..etape-2
git diff --name-status etape-2..etape-3

4.2. Commandes pour téléchargement sélectif

Extraction d'une étape

# Méthode 1: Archive vers dossier
git archive [commit-hash] | tar -x -C ../nom-dossier/

# Méthode 2: Archive vers ZIP
git archive [commit-hash] --format=zip > etape-X.zip

# Méthode 3: Shallow clone d'un commit spécifique
git clone --depth 1 https://github.com/inskillflow/ASPNETCORE-WEB-API-XtraWork.git
cd ASPNETCORE-WEB-API-XtraWork
git fetch --depth 1 origin [commit-hash]
git checkout [commit-hash]

Synchronisation et mise à jour

# Mettre à jour le repository local
git fetch origin

# Voir les nouveaux commits
git log HEAD..origin/main --oneline

# Mettre à jour vers la dernière version
git pull origin main

# Vérifier les branches locales
git branch -a

# Nettoyer les branches obsolètes
git remote prune origin
